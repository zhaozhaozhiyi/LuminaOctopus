# LuminaOctopus（拾光章鱼）品牌定义

版本：v0.1  
日期：2026-03-19  
适用范围：产品命名、官网与应用文案、发布物料、README/PRD/设计稿中的对外表述

---

## 1. 品牌基本信息

- **中文品牌名**：拾光章鱼  
- **英文品牌名**：LuminaOctopus（建议作为产品/开源仓库主名）  
- **标准写法**：
  - 中文：`拾光章鱼`
  - 英文：`LuminaOctopus`（不加空格，不使用 `Lumina Octopus` 作为正式品牌名；可在文案中按需拆分以增强可读性）
- **一句话定位**：把网站抓取为可离线浏览的“站点快照”，并可暂停恢复、回看历史、导出带走。  
  - English: Turn websites into offline-browsable snapshots—with pause/resume, job history, and export.

## 2. 品牌内核

### 2.1 品牌使命（Mission）

让重要的网页内容在任何时候都可被保存、回看与离线使用。  
English: Make important web content easy to save, revisit, and use offline—anytime.

### 2.2 品牌愿景（Vision）

成为个人与团队进行网站内容归档、迁移对照与离线阅读的默认工具。

### 2.3 品牌价值主张（Value Proposition）

- **离线可用**：生成可离线打开并可点击跳转的站点镜像（HTML 入口与关键资源本地化）。
- **可控可恢复**：抓取任务可暂停/恢复/停止；状态、队列与错误可解释；刷新/重启后历史仍可导出。
- **礼貌抓取**：强调同站点约束、并发/延迟控制与 robots 策略开关，尊重目标站点与资源。
- **清晰边界**：不承诺复杂登录/MFA、强交互 SPA 的完整离线复现；不做法证级取证归档能力。

## 3. 目标用户与典型场景

### 3.1 目标用户

- **文档/知识工作者**：离线阅读帮助中心、教程、课程资料；在网络不稳定/受限环境下使用。
- **研发/运维/内容团队**：迁移前备份站点、回归对照、环境隔离下的文档镜像与快照留存。
- **个人站长/设计师**：保存网站快照，留存视觉与内容，便于对比与复盘。

### 3.2 核心场景（我们必须表现好的时刻）

- 输入一个文档站首页 → 设置深度与策略 → 开始抓取 → 最终离线打开 `index.html` 可浏览主要页面与资源。
- 抓取过程中可暂停/恢复 → 即使刷新页面或服务重启，仍能在历史任务里找到并导出。
- 某些页面/资源失败 → 能从日志/状态中快速定位失败原因并继续整体任务。

## 4. 品牌人格与语气（Voice & Tone）

### 4.1 品牌人格（3 个关键词）

- **可靠**：可解释、可复现、可恢复，不玩“黑箱”。
- **克制**：强调边界与合规提示，不夸大“全站都能抓”。
- **利落**：以任务为中心，信息密度高但不啰嗦。

### 4.2 语气规范

- **面向用户**：短句、动词开头、明确结果（“导出 zip”“离线预览”“暂停后可恢复”）。
- **面向开发者/开源**：准确术语 + 清晰边界（same-site、depth、concurrency、delay、robots、jobId）。
- **避免**：夸张承诺、暗示绕过限制、引导不当使用的措辞。

## 5. 信息表达框架（Messaging）

### 5.1 三大信息支柱（对外主张）

1. **抓取成“离线可浏览”**：不是单页保存，而是可点击、可跳转的站点快照。  
2. **任务化与可恢复**：暂停/恢复/历史/导出，可靠地把结果带走。  
3. **可控与礼貌**：同站点、深度、并发、延迟、robots，可按场景调整。

### 5.2 Slogan（推荐）

- **中文**：把网页捞上来，变成你的资料库。  
- **英文**：Save sites for offline reading.

### 5.3 一句话电梯介绍（两种口径）

- **产品口径**：拾光章鱼把网站抓取为可离线浏览的整站镜像，并提供可暂停恢复、历史回看与导出。  
- **技术口径**：基于 Next.js 的 Web 版整站镜像工具，爬虫核心为纯 Node 引擎，便于后续 Electron 主进程复用。

## 6. 命名体系（产品内名 / 模块名）

### 6.1 核心对象命名（建议固定）

- **Job**：一次抓取任务（`jobId`）  
- **Session**：当前运行的抓取会话（active job）  
- **Queue / Items**：待处理与已处理条目列表（page/asset）  
- **Export**：导出（Web：zip；桌面端：落盘目录为主）  
- **Preview**：离线预览（仅限 job 的 site 目录范围）

### 6.2 页面/导航命名（建议）

- **Home**：Crawl（抓取）  
- **Jobs**：History（历史任务）  
- **Logs**：Logs（日志）  
- **Settings**：Settings（设置）  
- **Export**：Export（导出）

### 6.3 按钮与状态文案（推荐基线）

- **Primary**：开始抓取 / 暂停 / 恢复 / 停止 / 导出 zip  
- **状态**：idle / running / paused / stopped / done / error / skipped（UI 可本地化为：空闲/运行中/已暂停/已停止/已完成/失败/已跳过）

## 7. 视觉与符号方向（不绑定具体 UI）

### 7.1 视觉隐喻

- **“拾光”**：微光、时间、留存、快照、归档。
- **“章鱼”**：触手并发处理、抓取/捞取、在复杂网络里灵活探索。

### 7.2 Logo/图形建议（供设计参考）

- **形态**：简化章鱼轮廓 + 负形“光点/光束”或“页面角标”元素。
- **风格**：工具型产品优先“克制、清晰、可读”，避免过度卡通化。
- **动效（可选）**：触手轻微律动与“光点落入仓库/盒子”的过渡，用于加载/抓取进行中。

### 7.3 颜色方向（建议）

- **主色**：深海墨色（可靠、专业）  
- **强调色**：拾光暖金或偏琥珀（“光”与“收集”的提示）  
- **辅助**：低饱和灰蓝（信息层级与长列表可读性）

## 8. 合规与伦理提示（品牌底线）

拾光章鱼面向“备份、离线阅读、迁移对照与归档”场景。对外文案与产品提示需坚持：

- **尊重 robots 与站点策略**：提供开关与说明，不默认鼓励规避限制。
- **版权与授权提示**：强调用户需确保对抓取内容拥有使用权或获得授权。
- **不暗示“破解/绕过登录”**：不承诺复杂登录、MFA、强交互应用的完整离线复现。

## 9. 禁用表述（不要这么说）

- “保证任何网站都能完整离线”
- “一键抓取无限规模全网内容”
- “绕过登录/限制/付费墙”
- “法证级/可证明完整性归档”（除非未来明确实现并验证）

## 10. 示例文案（可直接复用）

### 10.1 官网/README（短）

- **标题**：拾光章鱼：把网站保存成离线可浏览的整站镜像  
- **副标题**：可暂停恢复、可回看历史、可导出带走；用可控的深度/并发/延迟，礼貌抓取你需要的内容。

### 10.2 产品内空态

- 输入入口 URL，选择抓取深度，点击“开始抓取”。  
- 抓取完成后，你可以离线预览或导出 zip。

### 10.3 设置提示（短）

- 并发越高越快，但更可能触发限流；建议从 2～3 开始。  
- 若站点较敏感，请开启“遵守 robots”，并增加请求延迟。

## 11. 与产品目标的对齐（供团队内部校验）

- **G1 可用离线镜像**：品牌承诺“离线可浏览的站点快照”。  
- **G2 可控与可恢复**：品牌强调 jobs、暂停/恢复、历史与导出可靠。  
- **G3 可演进**：品牌叙事支持 Web → Electron 的自然迁移，爬虫引擎与 UI 解耦可复用。
